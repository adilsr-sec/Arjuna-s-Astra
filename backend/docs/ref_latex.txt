	\justify \fontsize{12}{22} \selectfont {Secure communication in military and mission-critical environments demands not only strong encryption but also covert transmission methods that conceal the existence of sensitive information itself. This project presents ARJUNA’S ASTRA, a military-grade secure communication system that enables hidden transmission of encrypted audio messages within standard digital images using advanced steganographic and cryptographic techniques. The system combines audio transformation, hybrid encryption, and randomized image steganography to create an invisible and tamper-evident communication channel.

The proposed system converts audio messages into compact spectrogram-based representations inspired by analysis and resynthesis sound spectrograph techniques. The transformed payload is secured using a hybrid cryptographic framework involving AES-256-GCM for data encryption and RSA-2048 for secure session key exchange. To further enhance confidentiality, the encrypted payload is embedded into cover images using randomized Least Significant Bit (LSB) steganography driven by a shared secret seed, making detection through conventional steganalysis significantly difficult. The system also incorporates security features such as optional secondary password protection, role-based access control, one-time audio retrieval, mission telemetry logs, and a panic-driven Burn Protocol for compromised scenarios.

A web-based application is developed using Next.js and FastAPI, providing secure authentication, audio encoding and decoding workflows, image-based covert transmission, and real-time monitoring capabilities through a military-inspired command dashboard. The system is designed to maintain high visual fidelity of stego-images while ensuring reliable extraction and reconstruction of hidden messages. By integrating cryptography, steganography, and tactical security controls into a unified framework, ARJUNA’S ASTRA offers an innovative solution for covert communication in defense and high-security applications, while demonstrating the practical potential of secure invisible communication systems.}

\section{ About the Project}

\large\justify ARJUNA’S ASTRA is a secure covert communication system designed to transmit sensitive audio messages by hiding encrypted information inside digital images using cryptography and steganography. The project aims to overcome the limitations of conventional secure communication systems, where encrypted messages may reveal the existence of secret communication. The proposed system converts audio messages into spectrogram-based data, secures them using hybrid encryption with AES-256 and RSA-2048, and embeds the encrypted payload into images using randomized Least Significant Bit (LSB) steganography. The system also includes security features such as role-based access control, one-time retrieval, telemetry logging, and Burn Protocol for enhanced operational security. Developed as a web-based application using Next.js and FastAPI, ARJUNA’S ASTRA provides an efficient framework for invisible and secure communication in defense and mission-critical environments.

\section{ Objective and Scope of Project}

\large\justify The primary objective of ARJUNA’S ASTRA is to develop a secure communication platform capable of hiding sensitive audio information within digital images while ensuring confidentiality, integrity and covert transmission. The project aims to combine advanced cryptographic algorithms and steganographic methods to provide invisible communication that is resistant to interception and unauthorized access. 

The scope of the project includes the development and implementation of audio transformation techniques, hybrid encryption methods, randomized image embedding algorithms, and secure decoding modules integrated into a web-based platform. The project focuses on defense-oriented and mission-critical communication scenarios, while also demonstrating possible applications in intelligence systems, secure emergency communication and high-security information exchange.


\section{ Definition of Problem}

\large\justify Traditional secure communication systems rely mainly on encryption to protect sensitive information. Although encryption secures the content of messages, it does not conceal the existence of communication itself, making encrypted traffic vulnerable to surveillance, interception and traffic analysis. In military and mission-critical environments, even revealing the presence of secret communication can compromise operations.

Conventional steganographic systems also face challenges such as predictable embedding patterns, vulnerability to steganalysis and lack of integrated security controls. Existing systems often fail to provide a unified framework combining covert communication, strong encryption and operational safeguards. This project addresses these limitations by proposing a secure system that hides encrypted audio messages within ordinary images while providing multiple layers of protection against detection and compromise.



\section{ Significance of the Project}

\large\justify The significance of ARJUNA’S ASTRA lies in providing a secure and covert communication framework by combining cryptography and steganography within a single system. Unlike conventional communication methods that only protect message content, the proposed system also conceals the existence of communication, making it suitable for defense and mission-critical applications where operational secrecy is essential. By integrating hybrid encryption, randomized LSB embedding, role-based access control, one-time retrieval, telemetry logging and Burn Protocol, the system enhances confidentiality, resistance against detection and operational security. The project also demonstrates the practical integration of cybersecurity, signal processing and secure system design, while offering potential applications in defense, intelligence and other high-security communication environments.

\section{ Introduction to Arjuna's Asthra }

\large\justify ARJUNA’S ASTRA is an advanced covert communication system designed to provide secure transmission of sensitive audio messages by concealing encrypted data within digital images. The system combines cryptography, steganography and audio signal processing to create an invisible communication channel that protects both the content and existence of communication. It employs spectrogram-based audio encoding, hybrid encryption using AES and RSA, and randomized Least Significant Bit embedding to ensure confidentiality and resistance against detection. In addition to covert message transmission, the system incorporates security features such as role-based access control, one-time retrieval, telemetry logging and Burn Protocol to strengthen operational security. The project is developed as a web-based platform for defense and mission-critical environments, providing a practical framework for secure and hidden communication.


\section{ Traditional Image Forgery Detection Techniques }

\large\justify Traditional image forgery detection techniques are among the earliest methods developed to identify manipulated or altered images. These approaches primarily rely on analyzing low-level features such as pixel values, noise patterns, color distribution, and compression artifacts to detect inconsistencies within an image. Techniques like pixel-based analysis examine irregularities in noise, texture, and color that may arise when different image regions are edited or combined. Similarly, compression and artifact analysis focuses on identifying anomalies in formats such as JPEG, where inconsistent compression patterns can indicate tampering. While these methods were effective for detecting basic image manipulations, they are limited when applied to modern AI-generated images. Advanced generative models are capable of producing highly consistent and realistic outputs, making it difficult for traditional techniques to identify any visible or statistical discrepancies. As a result, these methods are considered less reliable in current scenarios and have largely been replaced by more advanced deep learning-based approaches.

\subsection{ Pixel-Based Analysis }

\large\justify Early image forgery detection techniques relied heavily on pixel-level analysis to identify inconsistencies within an image. These methods examine noise patterns, color distribution, and texture variations to detect signs of manipulation. For instance, noise inconsistencies arise when different parts of an image have been altered or combined from multiple sources, resulting in uneven noise levels. Similarly, irregularities in color and texture can indicate tampering, as edited regions may not perfectly match the natural characteristics of the original image. While these techniques were effective for detecting simple edits, they are less reliable when dealing with highly sophisticated synthetic images generated by modern AI models.

\subsection{ Compression and Artifact Analysis}
\large\justify Another traditional approach involves analyzing compression artifacts and statistical properties of images. JPEG compression, which is commonly used in digital images, introduces specific patterns and distortions that can be studied to identify manipulation. For example, inconsistencies in JPEG artifacts across different regions of an image may suggest that parts of the image were altered or recompressed separately. In addition, statistical irregularities in pixel distributions, frequency components, or image metadata can provide clues about image authenticity. These methods rely on detecting deviations from expected patterns in naturally captured images, making them useful for basic forgery detection.

\subsection{Limitations of Traditional Methods}
\large\justify Despite their initial effectiveness, traditional image forgery detection techniques have significant limitations in the context of modern AI-generated images. Advanced generative models are capable of producing highly realistic images that maintain consistent noise patterns, textures, and compression characteristics, making it difficult for these methods to detect any anomalies. Furthermore, these techniques often lack scalability and robustness when applied to large and diverse datasets. As a result, they struggle to generalize across different types of manipulations and fail to keep up with the rapid advancements in AI-based image generation technologies.



\section{Deep Learning Approaches (CNN-Based Methods)}



\subsection{Overview of CNNs}

\large\justify Convolutional Neural Networks (CNNs) are a fundamental class of deep learning models designed specifically for image processing and computer vision tasks. They operate by applying convolutional filters across input images to detect patterns such as edges, textures, and shapes. These filters move spatially across the image, enabling the network to learn spatial hierarchies of features. A typical CNN architecture consists of convolutional layers for feature extraction, pooling layers for reducing dimensionality, and fully connected layers for final classification. This layered structure allows CNNs to progressively transform raw pixel data into meaningful representations.

\large\justify Feature extraction in CNNs is performed automatically through hierarchical learning. In the initial layers, the network captures low-level features such as edges and corners. As the depth increases, intermediate layers identify textures and repetitive patterns, while deeper layers recognize high-level semantic structures such as objects and shapes. This ability to learn features directly from data eliminates the need for manual feature engineering, making CNNs highly effective for image classification tasks.


\subsection{CNNs for Deepfake and GAN Detection}
\large\justify CNNs have been extensively applied in detecting AI-generated images and deepfakes due to their strong capability in image classification. These models are trained on datasets containing both real and synthetic images, enabling them to learn distinguishing features between authentic and manipulated content. In the context of GAN-generated image detection, CNNs analyze inconsistencies such as unnatural textures, irregular lighting patterns, and pixel-level artifacts that often arise during image synthesis.

\large\justify Compared to traditional handcrafted methods, CNN-based approaches provide significant improvements in performance. They achieve higher accuracy by automatically learning discriminative features and adapting to complex data distributions. As highlighted in the study, CNN models demonstrated substantial advancements over earlier statistical techniques in detecting fake images, making them a dominant approach in digital image forensics .



\subsection{Limitations of CNNs}
\large\justify Despite their effectiveness, CNNs have inherent limitations when applied to advanced AI-generated image detection. One of the major challenges is their limited ability to capture global context within an image. Since CNNs rely on local receptive fields, they primarily focus on nearby pixel relationships and struggle to understand long-range dependencies across different regions of an image.

\large\justify Additionally, CNNs face difficulties in detecting subtle manipulations introduced by modern generative models. As GANs become more sophisticated and produce highly realistic images, the differences between real and synthetic images become increasingly fine-grained. CNNs often fail to capture these subtle inconsistencies, leading to reduced performance in complex scenarios. The paper emphasizes that the effectiveness of CNNs decreases as generative models evolve, highlighting the need for more advanced architectures .



\section{Transformer-Based Models in Computer Vision}
\large\justify Transformers were originally developed in the field of Natural Language Processing (NLP), where they revolutionized sequence modeling by treating text as a sequence of tokens. Their success in capturing contextual relationships inspired their adoption in computer vision tasks. Unlike traditional models, transformers do not rely on sequential processing but instead utilize a mechanism that allows parallel computation and global information exchange.

\large\justify The core component of transformers is the self-attention mechanism, which enables the model to assign importance to different parts of the input data. Through self-attention, the model can learn relationships between distant elements within the input, making it highly effective in capturing global dependencies. This characteristic is particularly valuable in tasks where understanding the overall structure and context is crucial.

\subsection{Adaptation to Vision Tasks}
\large\justify In computer vision, transformers are adapted by representing images as sequences of smaller patches. Each image is divided into fixed-size patches, and each patch is treated as a token similar to words in a sentence. These tokens are then processed by the transformer model, allowing it to analyze the entire image in a holistic manner.

\large\justify This approach enables global feature learning, which is a significant advantage over CNNs. Instead of focusing only on local regions, transformer-based models consider relationships between all parts of the image simultaneously. As a result, they are better equipped to capture complex patterns and subtle variations, making them highly suitable for challenging tasks such as AI-generated image detection.

\section{Vision Transformers (ViTs) for Image Detection}
\subsection{Architecture of ViT}
\large\justify The Vision Transformer (ViT) introduces a novel architecture for image classification by treating images as sequences of patches. Initially, the input image is divided into non-overlapping patches, which are then flattened and transformed into vector representations through a process known as patch embedding. To retain spatial information, positional encoding is added to these embeddings, ensuring that the model understands the relative positions of patches within the image.

\large\justify These embedded patches are then passed through a transformer encoder composed of multiple layers of multi-head self-attention and feed-forward networks. This structure allows the model to capture complex relationships and long-range dependencies across different regions of the image. Finally, a classification token is used to represent the entire image, and its output is fed into a fully connected layer to produce the final prediction. The architecture illustrated in the paper demonstrates how this pipeline effectively processes image data for classification tasks.

\subsection {Advantages of ViTs}

\large\justify Vision Transformers offer several advantages over traditional CNN-based models, particularly in capturing long-range dependencies within images. By leveraging self-attention, ViTs can analyze relationships between distant regions, which is essential for identifying subtle inconsistencies in AI-generated images.

\large\justify Furthermore, ViTs have demonstrated superior performance in complex image classification tasks. The study reports that the ViT model achieves high accuracy, reaching approximately 98.2\%, and consistently outperforms CNN-based approaches. This improved performance highlights the effectiveness of transformer-based architectures in handling intricate visual patterns and detecting synthetic content.

\subsection {Applications in AI-Generated Image Detection}

\large\justify Recent research has shown that Vision Transformers are highly effective in detecting AI-generated images. By leveraging their ability to learn global features and complex patterns, ViTs can distinguish between real and synthetic images with high precision. The use of pretrained weights and data augmentation further enhances their performance and generalization capabilities.

\large\justify When compared to CNN models, ViTs demonstrate clear advantages in terms of accuracy and recall. They are particularly effective in minimizing false negatives, which is critical in applications such as deepfake detection. The findings from the study confirm that ViTs provide a robust framework for image forensics and represent a promising direction for future research.

\section{Comparative Analysis of Existing Methods}

\large\justify A comparative analysis of traditional methods, CNN-based approaches, and Vision Transformers reveals a clear progression in performance and capability. Traditional methods rely on handcrafted features and statistical analysis, which often lack robustness and fail to generalize across diverse datasets. CNNs improve upon this by automatically learning hierarchical features, but their focus on local regions limits their ability to capture global context.

\large\justify Vision Transformers, on the other hand, introduce a paradigm shift by leveraging self-attention mechanisms to model global relationships within images. As reported in the study, ViTs achieve higher accuracy compared to CNN-based models, reaching approximately 98.2\%, while CNNs typically range between 91\% and 96\% . However, this improved performance comes at the cost of increased computational complexity, as ViTs require more resources and longer training times.

\large\justify Each approach has its strengths and weaknesses. Traditional methods are computationally efficient but lack accuracy, CNNs offer a balance between efficiency and performance, while ViTs provide superior accuracy and global understanding at the expense of higher computational requirements. The choice of method depends on the specific application and available resources.

\section{Research Gaps}

\large\justify Despite significant advancements in AI-generated image detection, several research gaps remain. One major limitation is the inability of existing models, particularly CNNs, to effectively detect highly sophisticated GAN-generated images. As generative models continue to improve, detection methods must evolve to keep pace with increasingly realistic synthetic content.

\large\justify Another critical challenge is the high computational cost associated with Vision Transformers. These models require substantial computational resources, making them difficult to deploy in real-world scenarios with limited hardware capabilities. This constraint highlights the need for more efficient and lightweight architectures.

\large\justify Additionally, current detection systems are not well-suited for real-time applications. The complexity and processing requirements of advanced models hinder their ability to perform instant detection, which is essential in many practical use cases. Furthermore, models often struggle with generalization across different datasets, performing well on trained data but failing to adapt to unseen data distributions.

\large\justify Addressing these gaps requires future research to focus on improving model efficiency, enhancing generalization capabilities, and developing real-time detection systems. Such advancements will be crucial for ensuring the reliability and scalability of AI-generated image detection technologies.

With the advancement of artificial intelligence, many models can create highly realistic images that are difficult to distinguish from real images. \section{Existing System Disadvantages}
\large\justify Existing systems used for detecting AI-generated images mainly rely on traditional image analysis techniques and Convolutional Neural Network (CNN) based models. Early detection methods focused on identifying inconsistencies in pixel values, lighting patterns, or compression artifacts present in manipulated images. However, with the rapid advancement of artificial intelligence and generative models such as Generative Adversarial Networks (GANs), these traditional methods have become less effective. Modern AI models are capable of generating highly realistic images that closely resemble real photographs, making it extremely difficult for traditional techniques to distinguish between real and synthetic content. CNN-based models improved detection performance compared to earlier approaches, but they still have certain limitations. CNNs mainly focus on local features of images and may fail to capture global relationships between different regions of the image. As a result, subtle differences between real and AI-generated images may not be properly detected. Another disadvantage of existing systems is the higher probability of false predictions, where real images may be incorrectly classified as fake or AI-generated images may remain undetected. In addition, many existing systems struggle to handle large and diverse datasets efficiently, which limits their scalability and reliability in real-world applications.

\section{Proposed System Advantages}
\large\justify The proposed system addresses the limitations of existing methods by using a Vision Transformer (ViT) based architecture for detecting AI-generated images. Vision Transformers introduce a new approach to image analysis by dividing images into smaller patches and processing them using a transformer architecture with self-attention mechanisms. This allows the model to capture both local and global relationships within the image more effectively. One of the main advantages of the proposed system is its ability to detect subtle patterns and complex structures that differentiate real images from AI-generated images. The self-attention mechanism enables the model to analyze the relationships between different regions of an image, improving the accuracy of detection. Another advantage is improved generalization capability due to the use of data augmentation techniques such as image rotation, cropping, and flipping during training. The system also benefits from pretrained weights obtained from large-scale datasets, which helps the model learn meaningful visual features more efficiently. As a result, the proposed system achieves higher detection accuracy and performs better than traditional CNN-based models. Additionally, the system is scalable and capable of handling large datasets, making it suitable for real-world applications such as digital media verification and cybersecurity.

\section{Methodology}
\large\justify The methodology used in this project involves several important steps. The first step is collecting a dataset containing both real images and AI-generated images. \large\justify Next, the images are divided into fixed-size patches and converted into embeddings. The model is trained using labeled data to learn the distinguishing features of real and AI-generated images.

\section{Vision Transformer Architecture}
\subsection{Dataset Selection and Preprocessing}
\large\justify The first step in implementing the Vision Transformer (ViT) architecture involved selecting and preparing an appropriate dataset for training and evaluation. The dataset used in this project was sourced from the Kaggle Digital Image Forensics repository, consisting of a balanced collection of real and AI-generated images. The dataset included a total of 30,000 images, with 15,000 real images collected from various online sources and 15,000 AI-generated images created using Generative Adversarial Networks (GANs). Each image was labeled as either “real” or “fake,” making it suitable for binary classification.
\large\justify To enhance model performance and ensure compatibility with the Vision Transformer, several preprocessing steps were applied. All images were resized to a fixed dimension of 224 × 224 pixels, which is the standard input size required by the ViT model. Pixel values were normalized to the range [0,1] to improve training stability and convergence.
\large\justify In addition, data augmentation techniques were employed to increase the diversity of the training data and improve the model’s generalization capability. These techniques included random cropping, horizontal flipping, rotation, and color jittering. Furthermore, the images were divided into smaller non-overlapping patches of size 16 × 16 pixels, and each patch was flattened into a vector representation. Positional encoding was then added to these patch embeddings to preserve spatial relationships within the image.
\subsection{Dataset Structure}
\large\justify The dataset used for training the Vision Transformer model was organized into two main categories: real images and AI-generated images. Each category contained an equal number of samples to ensure a balanced dataset, which helps prevent bias during model training.
\large\justify The dataset was structured in a way that facilitates efficient training and evaluation. It was divided into training and validation sets, typically in an 80:20 ratio. The training set was used to train the model, while the validation set was used to evaluate its performance and tune hyperparameters. Each image in the dataset was associated with a corresponding label indicating whether it was real or fake.
\large\justify This structured approach enabled the model to learn distinguishing features between authentic and AI-generated images effectively, while also providing a reliable framework for performance evaluation.
\subsection{Model Training and Evaluation}
\large\justify Once the dataset was prepared, the Vision Transformer model was trained using the preprocessed image data. The model operates by analyzing sequences of image patches and learning complex patterns that distinguish real images from AI-generated ones.
 \large\justify The training process involved initializing the model with pretrained weights to leverage previously learned visual features, which significantly improved performance and reduced training time. The model was trained using the Adam optimizer with a learning rate of 1e-4. A binary cross-entropy loss function was used, as the task involves binary classification between real and fake images.
\large\justify The training was conducted with a batch size of 32 over 50 epochs. Early stopping was implemented to prevent overfitting by monitoring the validation loss. During training, the model continuously adjusted its internal parameters to improve prediction accuracy.
\large\justify For evaluation, several performance metrics were used, including accuracy, precision, recall, F1-score, and ROC-AUC. A confusion matrix was also generated to analyze the model’s classification performance in detail, including true positives, false positives, true negatives, and false negatives.
\subsection{Technical Side}
\large\justify From a technical perspective, the Vision Transformer model processes images differently from traditional convolutional neural networks. Instead of analyzing the entire image at once, the image is divided into smaller patches, which are treated as a sequence of tokens.
\large\justify Each patch is embedded into a vector and combined with positional encodings to retain spatial information. These embeddings are then passed through a transformer encoder consisting of multiple layers of multi-head self-attention mechanisms and feed-forward neural networks. In this implementation, the encoder contains 12 layers with 8 attention heads, allowing the model to capture both local and global dependencies within the image.
\large\justify A special classification token ([CLS]) is added to the sequence of patch embeddings. After processing through the transformer layers, the output corresponding to this token is used as a representation of the entire image. Dropout regularization (ranging from 0.1 to 0.2) is applied to reduce overfitting and improve generalization.
\subsection{Integration of Vision Transformer Model}
\large\justify The trained Vision Transformer model was integrated into the overall system to enable real-time detection of AI-generated images. The integration involved developing interfaces that allow input images to be processed through the trained model and generate predictions.
\large\justify When a user provides an image, it is first preprocessed using the same steps applied during training. The processed image is then passed through the Vision Transformer model, which outputs a probability score indicating whether the image is real or AI-generated. Based on this output, the system classifies the image accordingly.
\large\justify This integration ensures a seamless workflow from image input to prediction output, making the system suitable for practical applications such as digital image verification and deepfake detection.


\section{Grad-CAM Based Explainability Module}
\subsection{Introduction to Grad-CAM}
\large\justify To enhance the interpretability of the Vision Transformer model, Gradient-weighted Class Activation Mapping (Grad-CAM) was incorporated into the system. Grad-CAM is a visualization technique used to highlight the important regions of an image that influence the model’s prediction for a specific class.
\large\justify In this project, Grad-CAM is used to provide insights into how the model differentiates between real and AI-generated images. Instead of treating the model as a black box, this technique allows users to understand which parts of the image contributed most to the final decision.

\subsection{Working Principle of Grad-CAM}
\large\justify Grad-CAM works by utilizing the gradients of the target class flowing into the final layers of the model. These gradients are used to compute importance weights for different feature maps, which are then combined to produce a localization map.
\large\justify The process involves the following steps:
\item [1.] Forward Pass:
The input image is passed through the trained model to obtain class scores.
\item [2.] Feature Map Extraction:
Activations are extracted from the final feature layer of the model.
\item [3.] Gradient Computation:
Gradients of the target class score are computed with respect to the extracted feature maps.
\item [4.] Global Average Pooling:
The gradients are averaged to obtain importance weights for each feature map.
\item  [5.] Feature Map Combination:
The feature maps are combined using the computed weights to generate a coarse localization map.
\item [6.] ReLU Activation:
A Rectified Linear Unit (ReLU) is applied to retain only the features that positively influence the prediction.
\end{itemize}

\subsection{Integration with Vision Transformer}
\large\justify Although Grad-CAM is traditionally used with Convolutional Neural Networks, it was adapted in this project to work with the Vision Transformer architecture. The attention-based features of the transformer were utilized to approximate spatial importance.
\large\justify The Grad-CAM module was integrated after the model prediction stage. Once the Vision Transformer classifies an image as real or fake, Grad-CAM generates an explanation map indicating the regions responsible for that classification..
\subsection{Advantages of Using Grad-CAM}
\large\justify The inclusion of Grad-CAM provides several benefits:
\large\justify The process involves the following steps:
\item [1.] Improves model transparency and interpretability
\item [2.] Helps in understanding model behavior
\item [3.] Assists in debugging incorrect predictions
\item [4.] Enhances user trust in the system


\section{Heatmap Visualization Module}
\subsection{Introduction to Heatmap Visualization}
\large\justify To provide a clear and intuitive understanding of the model’s decision-making process, a heatmap visualization module was incorporated into the system. The heatmap is generated based on the output of the Grad-CAM module and highlights the regions of the image that most strongly influenced the model’s prediction.
\large\justify This visualization helps users easily interpret whether the model is focusing on meaningful parts of the image while classifying it as real or AI-generated

\subsection{Heatmap Generation Process}
\large\justify The heatmap is created by transforming the output of the Grad-CAM module into a visual representation. The process involves the following steps:
\item [1.] Normalization:
The Grad-CAM output is normalized to a suitable range to ensure proper visualization.
\item [2.] Resizing:
The generated activation map is resized to match the original input image dimensions (224 × 224 pixels).
\item [3.] Color Mapping:
A color scheme (such as red, yellow, and blue) is applied to represent the intensity of important regions. Warmer colors (red/yellow) indicate higher importance, while cooler colors (blue) indicate lower importance.
\item [4.] Overlaying:
The heatmap is superimposed onto the original image, allowing users to visually correlate the highlighted regions with the actual image content.
\subsection{System Integration}
\large\justify The heatmap visualization module is integrated as the final stage of the system pipeline. After the Vision Transformer predicts whether an image is real or fake and the Grad-CAM module generates the importance map, the heatmap is produced and displayed to the user.

@tf.keras.utils.register_keras_serializable()
class PatchExtractor(layers.Layer):
    def __init__(self, patch_size=PATCH_SIZE, **kwargs):
        super().__init__(**kwargs)
        self.patch_size = patch_size

    def call(self, images):
        patches = tf.image.extract_patches(
            images=images,
            sizes=[1, self.patch_size, self.patch_size, 1],
            strides=[1, self.patch_size, self.patch_size, 1],
            rates=[1, 1, 1, 1],
            padding="VALID",
        )
        patch_dims = patches.shape[-1]
        patches = tf.reshape(patches, [-1, NUM_PATCHES, patch_dims])
        return patches

    def get_config(self):
        config = super().get_config()
        config.update({"patch_size": self.patch_size})
        return config


@tf.keras.utils.register_keras_serializable()
class PatchEncoder(layers.Layer):
    def __init__(self, num_patches=NUM_PATCHES, embed_dim=EMBED_DIM, **kwargs):
        super().__init__(**kwargs)
        self.num_patches = num_patches
        self.embed_dim = embed_dim
        self.projection = layers.Dense(embed_dim)
        self.position_embedding = layers.Embedding(
            input_dim=num_patches,
            output_dim=embed_dim
        )

def compute_fdiff(image_path):
    img = cv2.imread(image_path)
    if img is None:
        raise ValueError("Invalid image")

    img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    img = cv2.resize(img, (IMG_SIZE, IMG_SIZE))

    encode_param = [int(cv2.IMWRITE_JPEG_QUALITY), JPEG_QUALITY]
    _, enc = cv2.imencode(".jpg", img, encode_param)
    img_comp = cv2.imdecode(enc, 1)
    img_comp = cv2.cvtColor(img_comp, cv2.COLOR_BGR2RGB)

    fdiff = cv2.absdiff(img, img_comp)
    fdiff = fdiff.astype("float32") / 255.0

def compute_fdiff(image_path, jpeg_quality=90):
    img = cv2.imread(image_path)
    if img is None:
        raise ValueError("Invalid image path")

    img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    img = cv2.resize(img, (IMG_SIZE, IMG_SIZE))

    encode_param = [int(cv2.IMWRITE_JPEG_QUALITY), jpeg_quality]
    _, enc = cv2.imencode(".jpg", img, encode_param)
    img_comp = cv2.imdecode(enc, 1)
    img_comp = cv2.cvtColor(img_comp, cv2.COLOR_BGR2RGB)

    fdiff = cv2.absdiff(img, img_comp)
    fdiff = fdiff.astype("float32") / 255.0
    return img, fdiff
